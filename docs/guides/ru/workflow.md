# Система автоматизации MCP Workflow

## Обзор

MCP Workflow — это фреймворк автоматизации Minecraft на основе YAML, заменяющий предыдущий `run.py` на основе аргументов командной строки.

**Основная философия проектирования: Preview Click (предварительный просмотр клика)** — перед выполнением реального клика сначала рисуется красный маркер на скриншоте, позиция подтверждается визуально, затем выполняется клик. Это исключает слепое угадывание координат.

## Быстрый старт

```bash
# Холостой прогон (без запуска MC, только проверка разбора YAML и логики шагов)
python scripts/run_yaml.py workflows/smoke_test.yaml --dry-run --skip-setup

# Полный запуск (автоматический запуск MC + выполнение всех шагов)
python scripts/run_yaml.py workflows/smoke_test.yaml

# Пропустить фазу запуска, если MC уже работает
python scripts/run_yaml.py workflows/smoke_test.yaml --skip-setup

# Выполнить только шаг 5
python scripts/run_yaml.py workflows/smoke_test.yaml --step 5

# Отключить встраивание контейнера
python scripts/run_yaml.py workflows/smoke_test.yaml --no-container
```

## Процесс Preview Click

Это ключевая инновация системы. Традиционный подход напрямую отправляет координаты для клика; если координаты неверны, клик бесполезен. Процесс предварительного просмотра состоит из двух этапов:

```
Шаг 1: preview_click    → Ставит в очередь красный маркер на (x,y), не выполняет никаких действий
Шаг 2: screenshot       → Скриншот + рисует красную точку/перекрестие/текст координат на изображении
         ↓ ИИ проверяет аннотированный скриншот ↓
Шаг 3: click            → После подтверждения правильности координат выполняет реальный клик
```

### Эффекты аннотации красной точки

- **Красный круг** (настраиваемый радиус, по умолчанию 10px)
- **Перекрестие** (горизонтальная + вертикальная линии)
- **Метка координат** `(426,236) Одиночная игра` с полупрозрачным чёрным фоном

### Пример YAML

```yaml
# Шаг 1: Предварительный просмотр координат
- action: preview_click
  x: 426
  y: 236
  label: "Одиночная игра"
  radius: 10
  color: "#FF0000"
  comment: "Предпросмотр: нарисовать красную точку на кнопке Одиночная игра"

# Шаг 2: Скриншот (автоматически будет включать красную точку)
- action: screenshot
  name: "preview_singleplayer"
  comment: "Проверка ИИ: красная точка находится на кнопке?"

# Шаг 3: Клик после подтверждения
- action: click
  x: 426
  y: 236
  comment: "Координаты подтверждены, выполнить клик"
```

## Поддерживаемые действия

| Действие | Параметры | Описание |
|------|------|------|
| `wait` | `seconds` | Ожидание указанного количества секунд |
| `screenshot` | `name` | Сделать и сохранить скриншот (автоматически рисует поставленные в очередь маркеры предпросмотра) |
| **`preview_click`** | `x, y, label, radius, color` | **Поставить в очередь красный маркер, клик не выполняется** |
| `click` | `x, y` | Выполнить клик по координатам |
| `click_btn_idx` | `index` | Клик по кнопке по индексу виджета |
| `click_btn_id` | `button_id` | Клик по кнопке по ID |
| `ctrl_on` | - | Войти в режим управления MCP (мышь отвязана) |
| `ctrl_off` | - | Выйти из режима управления |
| `key` | `key` | Нажать клавишу (например, `Escape`, `E`) |
| `paste` | `text, press_enter` | Вставить текст (обходит проблемы IME) |
| `scroll` | `clicks` | Прокрутка колёсика мыши |
| `look_delta` | `dyaw, dpitch` | Поворот угла обзора в игре (не двигает мышь) |
| `set_view_angle` | `yaw, pitch` | Установить абсолютный угол обзора |
| `right_click` | - | Правый клик |
| `enumerate_widgets` | - | Перечислить все виджеты на текущем экране |
| `get_screen_buttons` | - | Получить список кнопок текущего экрана |
| `cmd` | `command` | Выполнить команду MC (например, `/gamemode creative`) |
| `vision_check` | `prompt, expect, store_as` | Визуальный анализ скриншота ИИ |

## Формат YAML-рабочего процесса

```yaml
name: "Название рабочего процесса"
description: |
  Многострочное описание, объясняющее, что делает этот рабочий процесс и что он проверяет.

setup:
  version: "1.21.7-forge-57.0.2"   # Версия MC
  container: false                   # Использовать ли окно контейнера
  wait_after_connect: 15             # Секунд ожидания загрузки MC после подключения

steps:
  # Каждый шаг — это действие, выполняемое по порядку
  - action: wait
    seconds: 15
    comment: "Ожидание завершения загрузки MC"

  - action: ctrl_on
    comment: "Войти в режим управления"

  - action: screenshot
    name: "baseline"
    comment: "Базовый скриншот"
```

## Параметры конфигурации setup

| Поле | Тип | По умолчанию | Описание |
|------|------|--------|------|
| `version` | string | `"1.21.7-forge-57.0.2"` | Идентификатор версии MC |
| `container` | bool | `true` | Использовать ли встраивание контейнера Win32 |
| `wait_after_connect` | int | `15` | Секунд ожидания после подключения мода (пока не исчезнет заставка Mojang) |

## Аргументы CLI

| Аргумент | Описание |
|------|------|
| `<workflow.yaml>` | Путь к файлу YAML рабочего процесса (обязательно) |
| `--dry-run` | Режим холостого прогона, не отправляет реальные команды в MC |
| `--skip-setup` | Пропустить запуск MC (использовать, если MC уже запущен) |
| `--step N` | Выполнить только шаг N (индексация с 1) |
| `--interactive` | Пауза после каждого шага с ожиданием подтверждения |
| `--no-container` | Отключить окно контейнера |

## Структура файлов

```
minecraft-neoforge-mcp/
├── workflows/                        # Определения рабочих процессов YAML

├── scripts/

│   ├── workflow_engine.py            # Основной движок (выполнение действий, управление состоянием, аннотация скриншотов)

```

## Отличия от старого run.py

| Возможность | run.py (старый) | run_yaml.py (новый) |
|------|-------------|-----------------|
| Формат | Аргументы CLI | Файл YAML |
| Повторное использование | Низкое (каждый раз вводить аргументы) | Высокое (YAML под версионным контролем, можно делиться) |
| Preview click | Не поддерживается | ✅ preview_click + аннотация скриншотов |
| Структурированные комментарии | Нет | Каждый шаг имеет поле comment |
| Восстановление после ошибок | Нет | Отслеживание состояния success/error по шагам |
| Условное ветвление | Не поддерживается | vision_check / if_screen |

## Создание пользовательских рабочих процессов

1. Скопируйте `workflows/smoke_test.yaml` в качестве шаблона
2. Измените `name`, `description` и `steps`
3. Сначала проверьте синтаксис с помощью `--dry-run --skip-setup`
4. Протестируйте на реальном экземпляре MC

### Типичный шаблон: Найти кнопку и кликнуть

```yaml
# 1. Сначала сделайте скриншот текущего экрана
- action: screenshot
  name: "current_screen"

# 2. Используйте ИИ-зрение для поиска координат кнопки (или прочитайте вручную из результатов enum)
- action: vision_check
  prompt: "Найти координаты центра кнопки «Создать новый мир» в пикселях"
  store_as: "create_world_pos"

# 3. Предпросмотр этих координат
- action: preview_click
  x: "${variables.create_world_pos.x}"   # TODO: ссылка на переменную будет реализована
  y: "${variables.create_world_pos.y}"
  label: "Создать новый мир"

# 4. Скриншот для проверки
- action: screenshot
  name: "preview_create_world"

# 5. Подтвердить и кликнуть
- action: click
  x: 350
  y: 420
```

## Важные замечания

1. **Не входите в режим управления во время заставки Mojang** — это приведёт к зависанию игры на логотипе. `setup.wait_after_connect` должен быть ≥ 15 секунд.
2. **Для скриншотов предпочитайте Robot (AWT)** вместо нативных скриншотов MC — нативные скриншоты MC могут возвращать кэшированные кадры.
3. **Обеспечьте достаточное ожидание после каждого клика** — переходы GUI требуют времени, обычно 3–8 секунд.
4. **Маркеры preview_click рисуются при следующем screenshot** — это механизм очереди, а не мгновенный эффект.
