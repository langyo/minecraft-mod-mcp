# Minecraft 26.1.2 Forge Injection Principle

[English](26.1.2+forge.md) | [中文](../zh-CN/26.1.2+forge.md)

## Overview

MCP Mod for Minecraft 26.1.2 Forge uses the **Forge Event Bus** with the **new static BUS API**. This is the newest Forge era (ForgeGradle 7+) which introduces static event bus references (e.g., `ScreenEvent.Opening.BUS`) replacing the deprecated `FMLJavaModLoadingContext` approach. This represents a significant API modernization in the injection pipeline.


**Key change**: Each event class now defines its own static `BUS` field. Instead of one global `MinecraftForge.EVENT_BUS`, events are registered directly on their specific bus. This allows:
1. Better compile-time type safety
2. Easier IDE support (autocomplete on `.BUS`)
3. No dependency on `MinecraftForge.EVENT_BUS` for these events
4. Listener return types matter — `boolean` return can cancel the event (new pattern)

### Boolean Return Pattern

In the new API, some BUS listeners return a boolean instead of calling `event.setCanceled(true)`:

```java
// Old API (1.20.6):
MinecraftForge.EVENT_BUS.addListener((ScreenEvent.MouseButtonPressed.Pre event) -> {
    if (controlMode) {
        event.setCanceled(true);  // Side-effect
    }
});

// New API (26.1.2):
ScreenEvent.MouseButtonPressed.Pre.BUS.addListener(event -> {
    if (controlMode) {
        return true;  // Return value signals cancellation
    }
    return false;
});
```

## GLFW Mouse Callback Interception

**NOT USED in this version!** Instead of GLFW callback hijacking, this version uses a different strategy: keyboard input interception via `InputEvent.Key.BUS` with reflective `setCanceled()`. Mouse input is handled purely through the Forge event bus (`InputEvent.MouseButton.Pre.BUS`) without GLFW-level callbacks.

```java
        // Find widest button, split, add MCP button
        event.addListener(transferBtn);  // Modern API uses addListener()
    }
});
```

## Version-Specific Details

Forge 64.0.8, ForgeGradle 7.0.23+, Java 25. Uses **non-official mappings** (unlike 1.21.7 which uses official mappings). The `GuiGraphicsExtractor` class replaces `GuiGraphics` for rendering — it provides a `getScissorStack()` method for manual scissor region management. Keyboard events are intercepted via `InputEvent.Key.BUS` with reflective `setCanceled()` calling. Chat messages use `addMessage()` via reflection (since the API signature differed with non-official mappings). 280 lines in ModDevMcpMod.

## Key Differences Summary

| Feature | Old Forge (1.20.6) | New Forge (26.1.2+) |
|---------|-------------------|---------------------|
| Event registration | `MinecraftForge.EVENT_BUS.addListener()` | `EventClass.SubEvent.BUS.addListener()` |
| Mod lifecycle | `FMLJavaModLoadingContext.get()` | Not used (deprecated) |
| Cancel pattern | `event.setCanceled(true)` | `return true` (boolean) |
| Pause screen blocking | `event.setCanceled(true)` + manual screen close | `event.setNewScreen(null)` |
| ClickEvent | `ClickEvent.Action.OPEN_URL` | `ClickEvent.OpenUrl(URI)` |
